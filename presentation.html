Slide 5 — My original contribution

Location-Invariant Disease Detection

The novel piece no crop-scanner does: a model trained to stay accurate in any agro-ecological location — with no GPS, no zone, no location input. The farmer just takes a photo.

The same disease on the same crop looks different across East Africa's zones — and a model that memorises one location's look silently fails elsewhere:

  • Highland → cool, bright light; higher contrast & saturation
  • Lowland → warm light, hazy air, lower contrast
  • Humid zones → soft haze, dew glare, washed-out colour
  • Dry / dusty zones → warm cast, camera noise, desaturation

This is a domain-shift problem. I strengthen the vision model so it recognises the disease, not the location's appearance — no external data needed at scan time.

Slide 6 — How it works

Trained to See Through Location

1) AGRO-ECOLOGICAL STYLE AUGMENTATION each leaf re-rendered as many "locations" (white balance, contrast, saturation, haze, noise) 2) MIXSTYLE (Zhou et al., ICLR 2021) — inside MobileNetV2 mixes feature statistics across the batch -> synthesises UNSEEN locations, 0 extra parameters -> disabled at inference: deployed model = plain MobileNetV2 Model selection = balance(clean accuracy, WORST-location accuracy)

System Architecture

Flutter Mobile App (Scan / Farms / Weather / Profile) SQLite offline cache · English + Kinyarwanda | REST API (JWT) v FastAPI Backend :8001 POST /analyze/image (photo + crop only) | +--------+--------+----------------+ v v v v MongoDB Vision CNN (MobileNetV2) Open-Meteo users location-invariant weather + farms 20 classes elevation diagnoses (MixStyle + agro aug) (farm advisories only)

Diagnosis needs no location — the model is location-invariant. Farm GPS drives only weather & advisories.

Crop-aware inference: probabilities filtered by selected crop — a cassava scan cannot return a tomato disease.

Machine Learning Model

Architecture

  • MobileNetV2 (ImageNet transfer learning)
  • Custom classifier head → 20 classes
  • Input: 224×224 RGB leaf images
  • Augmentation: crop, flip, rotation, color jitter

Training: 80/20 split, class-weighted loss, checkpoint saved to kulimaiq_mobilenet.pth
